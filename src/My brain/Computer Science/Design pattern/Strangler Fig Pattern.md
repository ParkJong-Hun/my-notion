# Strangler Fig Pattern

<aside>
💡

마틴 파울러가 도입한 무화과에서 영감을 얻은 패턴.
모놀리식 앱을 마이크로서비스로 전환.

변환, 공존, 제거라는 세 단계로 구성.

</aside>

**변환**

기존 앱과는 병렬로 포팅하거나 재작성해 현대화된 컴포넌트를 식별하고 생성.

**공존**

모놀리식 앱을 롤백에 대비해 보관.

HTTP 프록시를 통해 외부 시스템 직접 호출을 가로채 트래픽을 현대화된 버전으로 리디렉션.

**제거**

트래픽이 기존 모놀리스에서 현대화된 서비스로 리디렉션되었으면 모놀리스에서 기존 기능 사용 중지.
